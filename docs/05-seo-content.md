## SEO METADATA STANDARDS

All 36+ pages must follow strict metadata guidelines for Yoast SEO and featured snippets.

### Meta Title Rules
- **Length:** 50–60 characters (optimal for SERPs)
- **Formula:** `Primary Keyword | Brand Name` OR `Unique Title (Keyword)`
- **Never:** Keyword stuffing, click-baiting, misleading claims

**Examples:**
- ✅ "Chilled Food Storage Solutions | ARCOPAN"
- ✅ "Industrial Blast Freezing Cabinets – Cold Storage"
- ❌ "Chilled Food Storage Chilled Food Solutions Cold" (stuffing)
- ❌ "The ONLY Freezer You'll Ever Need" (hyperbole)

### Meta Description Rules
- **Length:** 145–158 characters (optimal for SERPs)
- **Format:** Benefit + Evidence + CTA (optional)
- **Never:** Duplicate descriptions, generic text, click-baiting

**Formula:**
```
[Benefit] for [target audience]. [Proof/Evidence]. [CTA: "Learn more", "Explore solutions"].
```

**Example:**
```
"Blast freezing systems achieving -40°C in 90 min for meat processors. 
ISO 22000 certified. Reduce waste by 30%. Explore our models."
(154 chars)
```

### H1 Heading Rules
- **One per page:** Exactly one `<h1>` (enforced in content markdown)
- **Must match meta title:** Keyword alignment critical
- **Length:** 6–12 words (scannable)
- **Format:** `Primary Keyword + Modifier`

**Examples:**
- "Chilled Food Storage Solutions for Dairy"
- "Blast Freezing Equipment for Meat Processing"
- "Industrial Cold Storage Across All Industries"

## INTERNAL LINKING STRATEGY

Internal links drive page authority and SEO visibility. Use this hub-spoke model:

### Hub Pages (High Authority)
These pages receive links from all other pages:
1. **Homepage** (`/`) — 36 outbound links (one per page)
2. **Products Hub** (`/products/`) — 13 outbound links (one per product)
3. **Solutions Hub** (`/solutions/`) — 6 outbound links (one per solution)
4. **Industries Hub** (`/industries/`) — 6 outbound links (one per industry)

### Spoke Pages (Supported by Hub)
1. **Product Pages** (13 pages) — Each links back to hub + related 2 products
2. **Solution Pages** (6 pages) — Each links back to hub + related industries (2×)
3. **Industry Pages** (6 pages) — Each links back to hub + relevant solutions (2×)
4. **Resource Pages** (4 pages) — Link to FAQ, related solutions

### Contextual Links (Content Context)
- Solution pages link to related industries (natural mentions)
- Industry pages link to relevant products in body text
- Product pages mention applicable solutions in context
- All pages link back to homepage (footer)

### Internal Link Anchor Text
**Rules:**
- Never use "click here" or "read more"
- Use keyword-rich anchor text (2–4 words)
- Exact match primary keyword (5% of links)
- Partial match secondary keywords (20% of links)
- Branded anchor "ARCOPAN" (10% of links)
- Natural phrases (65% of links)

**Examples:**
- ✅ "blast freezing solution" (keyword-rich)
- ✅ "explore our dairy solutions" (natural)
- ✅ "food storage requirements" (partial match)
- ❌ "click here" (generic)
- ❌ "more info" (generic)

### Internal Link Structure Map

```
Homepage
  ├─ /products/ (Products Hub)
  │   ├─ /products/chilled-cabinet/ → links to /solutions/chilled/ + 1 related product
  │   ├─ /products/blast-freezer/ → links to /solutions/blast-freezing/ + 1 related product
  │   └─ [11 more products with same pattern]
  ├─ /solutions/ (Solutions Hub)
  │   ├─ /solutions/chilled/ → links to 2 related industries
  │   ├─ /solutions/frozen/ → links to 2 related industries
  │   └─ [4 more solutions]
  ├─ /industries/ (Industries Hub)
  │   ├─ /industries/food-beverage/ → links to 2 relevant products + 2 solutions
  │   ├─ /industries/meat-poultry/ → links to 2 relevant products + 2 solutions
  │   └─ [4 more industries]
  ├─ /resources/ → links to 3 related solutions + FAQ
  ├─ /about/ → links to 3 key products
  ├─ /contact/ → subtle link to 1–2 solutions
  └─ /projects/ → links to 2–3 relevant industries
```

## SCHEMA MARKUP STRATEGY

All schema deployed via JSON-LD in footer (see `seo/schema-templates/`).

### Page-Level Schema
Every page outputs:
1. **Organization schema** (once per page, footer)
2. **Breadcrumb schema** (all inner pages)
3. **Page schema** (implicit, home only)

### Content-Specific Schema

| Content Type | Schema Type | Required Fields | Example |
|---|---|---|---|
| Product page | Product | name, image, description, price, aggregateRating | Product structured data for product cards in SERPs |
| Company page | Organization | name, logo, location, contact, URL | Organization card in SERPs |
| Industry page | BreadcrumbList | itemListElement, name, position | Breadcrumb in SERPs |
| FAQ page | FAQPage | mainEntity, acceptedAnswer | FAQ rich results in SERPs |

### Schema Testing
- [ ] Validate all schema at [schema.org validator](https://validator.schema.org/)
- [ ] Test SERPs data in Google Search Console (Enhancements tab)
- [ ] Monitor Rich Results report monthly

## SITEWIDE SEO CONFIGURATION

### Robots Meta Tags
All pages set via Yoast SEO settings:
```
<meta name="robots" content="index, follow, max-snippet:-1, max-image-preview:large, max-video-preview:-1">
```

### Canonical Tags
- All pages: self-referential canonical (e.g., page links to itself)
- Duplicate content: canonical points to preferred version
- Auto-generated by Yoast SEO

### Open Graph Tags (Social Sharing)
Every page must include:
```html
<meta property="og:title" content="{page title}">
<meta property="og:description" content="{meta description}">
<meta property="og:image" content="{featured image URL}">
<meta property="og:url" content="{page URL}">
<meta property="og:type" content="website">
```

### Language Tags (WPML)
WPML auto-generates hreflang for all translations:
```html
<link rel="alternate" href="https://arcopan.com/products/" hreflang="en">
<link rel="alternate" href="https://arcopan.de/produkte/" hreflang="de">
<link rel="alternate" href="https://arcopan.fr/produits/" hreflang="fr">
```
